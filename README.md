# PythonHiringAnalyzer

Скрипт анализирует вакансии с Habr Career и помогает собрать рыночную выжимку для подготовки сильного IT-резюме:

- показывает только востребованные роли в основном консольном топе
- собирает навыки, требования и зарплаты
- сохраняет полный массив ролей в `.txt` и `.xlsx`
- отдельно проверяет релевантность leadership-позиций, чтобы снизить шум от `AdOps`, `1C`, `marketing`, `BI` и похожих нерелевантных вакансий

## Что делает скрипт

Скрипт:

1. Ищет вакансии по набору заранее подготовленных ролей и синонимов на русском и английском.
2. Собирает карточки вакансий и дедуплицирует их по URL.
3. Загружает детали вакансий: описание, требования, навыки, зарплату.
4. Извлекает навыки из:
   - заголовка вакансии
   - текста требований и описания
   - тегов навыков на странице, если они есть
5. Для leadership-ролей делает дополнительную проверку релевантности.
6. Формирует:
   - консольный топ востребованных ролей
   - текстовый отчет `.txt`
   - Excel-отчет `.xlsx`

## Как определяется востребованность

В основной топ на экране попадают только роли, которые проходят рыночный фильтр:

- `оценка спроса >= 5`
- `проанализировано минимум 3 релевантные вакансии`

Все остальные роли не теряются:

- они все равно сохраняются в `.txt`
- они все равно сохраняются в `.xlsx`

Это сделано специально: экранный вывод остается полезным для быстрого анализа рынка, а выгрузки содержат полный массив данных.

## Как определяется релевантность

Для повышения достоверности скрипт использует несколько слоев фильтрации:

- фильтрация по заголовку вакансии
- исключающие ключевые слова для нерелевантных направлений
- проверка контекста по описанию, требованиям и навыкам
- проверка совпадений с role-specific `ATS/AI keywords`

Особенно это важно для ролей вроде:

- `Team Lead`
- `Tech Lead`
- `Engineering Manager`
- `Solution Architect`

Без такой фильтрации поиск часто подтягивает нерелевантные роли из смежных областей.

## Что находится на выходе

### Консоль

В консоль выводится только рыночный топ:

- роль
- оценка спроса
- количество реально проанализированных релевантных вакансий
- ключевые навыки
- зарплатная сводка
- примеры вакансий
- полные требования к кандидату

### TXT-отчет

В `.txt` попадают все рассчитанные роли:

- востребованные
- нишевые
- исключенные из основного экранного топа

### Excel-отчет

Формируется `.xlsx` с листами:

- `Сводка`
- `Вакансии`
- `Навыки`

#### Лист `Сводка`

Содержит:

- роль на русском и английском
- оценку спроса
- число проанализированных вакансий
- поисковые запросы
- разбивку по запросам
- сводку по зарплатам
- `ATS/AI keywords`
- подтвержденные рынком ключевики
- ключевики, которых не нашлось в текущей выборке

#### Лист `Вакансии`

Содержит:

- роль
- название вакансии
- компанию
- город
- зарплату
- навыки
- полные требования
- описание
- ссылку

#### Лист `Навыки`

Содержит плоскую частотную таблицу:

- роль
- навык
- частота

## Запуск

Проект лучше запускать через локальное виртуальное окружение:

```bash
.venv/bin/python main.py
```

Если запускать системным `python3`, может не хватить установленных зависимостей.

## Зависимости

Скрипт использует:

- `requests`
- `beautifulsoup4`

Если нужно установить их вручную:

```bash
.venv/bin/pip install requests beautifulsoup4
```

## Как использовать результат для резюме

Практический сценарий:

1. Смотри основной консольный топ и выбирай 2-3 действительно востребованные роли.
2. Открывай Excel-лист `Сводка` и смотри:
   - подтвержденные рынком навыки
   - частые `ATS/AI keywords`
   - зарплатный диапазон
3. Открывай лист `Вакансии` и смотри полные требования живых вакансий.
4. На основе этого собирай:
   - заголовок резюме
   - summary
   - стек
   - bullet points с опытом

## Ограничения

Важно понимать ограничения:

- источник один: `Habr Career`
- верстка сайта может измениться
- часть вакансий может быть скрыта, устарела или индексируется неравномерно
- оценка спроса строится по поисковой выдаче, а не по официальному API
- leadership-роли на рынке часто называются по-разному, поэтому даже после фильтрации возможен шум

Из-за этого скрипт лучше использовать как инструмент рыночной ориентации, а не как абсолютную статистику.

## Когда стоит донастроить роли

Имеет смысл вручную доработать `ROLE_CATALOG`, если ты хочешь:

- сузить анализ под конкретный стек
- выделить только backend/mobile/data/management
- добавить локальные русские формулировки ролей
- вынести отдельно AI/LLM-трек

## Идеи для следующего шага

Полезные доработки:

- отдельный лист `Рыночный топ`
- отдельный режим `leadership_only`
- генерация `resume targets` по каждой роли
- экспорт в `json`
- автоматическая выжимка "что добавить в резюме в первую очередь"
